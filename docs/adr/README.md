## Ratified decisions handoff (decision + rationale, for Claude Code)

- **ADR-0001** — *Decision:* model the domain as exactly three pure deterministic functions (F1 solar, F2 lunar, F3 Hebrew-calendar) emitting absolute instants, with `(φ, λ, h, t)` as the only natural inputs and couplings declared explicitly. *Rationale:* determinism + absolute outputs make the core oracle-testable, timezone-free, and offline-capable; elevation must be first-class for terrain horizons; molad (F3) is arithmetic and separate from observed moon (F2).
- **ADR-0002** — *Decision:* every stream/custom/stringency is a parameter over first-order outputs; the core exposes knobs and resolves none. *Rationale:* avoids a combinatorial explosion of "denomination engines," keeps the core small and auditable, and makes variation data-driven and testable — matching a market where the math is commodity.
- **ADR-0003** — *Decision:* compute on-device with an embeddable open-license library (KosherJava LGPL / Hebcal CC-BY *code*), fully offline through a multi-day chag+Shabbat, validated against a neutral oracle (Wolfram / observatory), depending on no competitor engine. *Rationale:* the engine is a commodity, so capture it for free and spend effort on the moat; depending on B.A. Timing/Chazon Shamayim (an engine-maker that also sells a competing board) would be a network and strategic liability; validate non-circularly.
- **ADR-0004** — *Decision:* deliver terrain-corrected visible sunrise via per-site `(azimuth → horizon-angle)` profiles computed at provisioning from Copernicus GLO-30 baseline + 1 m lidar overrides (USGS 3DEP / IGN RGE ALTI; Israel TODO; FABDEM commercial-only), composed offline on-device with F1 + refraction. *Rationale:* this is the one scarce, defensible capability; heavy DEM work belongs at provisioning, leaving the device offline-correct and cheap; build it rather than depend on ChaiTables/Sky-View.
- **ADR-0005** — *Decision:* nothing about correctness may depend on the network; the network is for management/content/ruleset updates only, and the device stays correct with zero connectivity across a full multi-day chag+Shabbat. *Rationale:* the board matters most when the network is down or unwanted; correctness must fail-safe while freshness/management may degrade gracefully; staleness is handled by versioning with a bounded (labeling-only) failure mode.
- **ADR-0006** — *Decision:* refraction and horizon optics are first-order physics inside F1, but the conventions (apparent vs geometric, refraction coefficient, sea-level vs terrain) are parameters; provisioning and runtime share one refraction model. *Rationale:* refraction defines the horizon-crossing instant so it must live in F1, but the choices are policy so they must be knobs — preventing both baking a convention into physics and exiling physics into a correction layer.
- **ADR-0007** — *Decision:* civil time/DST live outside the core; the core stores absolute instants, and wall-clock is a boundary-only label from a versioned IANA/DST ruleset that is never trusted as fixed. *Rationale:* civil time is a mutable, decree-driven labeling convention, not a domain function; keeping it out preserves determinism and bounds the failure mode — a stale ruleset can mis-label but never make a zman wrong.
- **ADR-0008** — *Decision:* build and own the primary F1/F2/F3 engine behind a pluggable engine interface; `zmanim-core` serves only as a build/test oracle and an optional customer-selectable alternative, never the foundation; engine-selection is a correctness-bearing knob handed to ADR-0002. *Rationale:* the correctness foundation must be ours, not a derivative-licensed dependency; relinkability is satisfied by architecture, not a license claim; two engines mean two correctness surfaces, so the no-drift guarantee holds only within one engine choice.
- **ADR-0009** — *Decision:* ratify the parameter-vector schema + F1/F2/F3 interface contract (`docs/spec/parameter-vector-and-interface-contract.md`): zmanim as a typed read-spec union (data, not code), terrain/refraction/engine-selection as knobs, the four 0001 couplings as typed dependencies; default `refraction.model = standard-atmospheric` with `meeus-noaa`/`halachic-fixed-coefficient` as alternatives; required-vs-optional cut and `base → tenant → site` preset precedence fixed. *Rationale:* a single data-parameterized core stays oracle-testable and auditable (0002/0003); the standard-atmospheric model composes with terrain horizon profiles where a fixed coefficient degrades; layering and defaults stay outside the core so it resolves none. Resolves 0002's schema question and closes 0006's refraction default; serialization (0008) and Israel DTM (0004) stay open.
- **ADR-0010** — *Decision:* the owned F1/F2/F3 engine is written in **Rust**, **freestanding `no_std`** on-device (org/0006 Profile A); one engine source compiled to every target (native + WASM) with a vendored deterministic FP math library; F1/F2 stay double-precision, F3 exact integer; LGPL relinkability preserved via a C-ABI FFI boundary to the oracle/alternative engine. *Rationale:* memory safety without a GC is the freestanding gap (C lacks it; managed languages are excluded for the engine by the no-resident-runtime rule; Zig is pre-1.0); single source → native + WASM serves one-core-no-drift (0008/0009); `no_std` fits the edge envelope and the cold-start invariant. Scope: engine only — management repos stay language-open. Closes open-decision #8's engine-language + runtime-model; serialization (0008) stays open.
- **ADR-0011** — *Decision:* the cross-repo param-vector + horizon-profile wire format is **CBOR** (RFC 8949), with a **CDDL** (RFC 8610) schema as the shared codegen-free contract and **COSE_Sign1** (RFC 9052) signing over deterministically-encoded CBOR; **two** independently-versioned, independently-signed artifacts (parameter vector; horizon-profile envelope = binding metadata + a packed azimuth→angle array). *Rationale:* the writer is provisioning-builder (not the Rust engine), so the contract must be cross-language by construction — a Rust-native format is exactly the in-ecosystem default 0008 forbade; CBOR has a Python writer / JS reader / `no_std` Rust reader; CDDL gives protobuf-like clarity without codegen lock-in; COSE folds org/0006 §7's signed-artifact requirement into the same decision; storing angles as f32/f16/fixed-point is input-data compactness, not a breach of the F1/F2 double-precision computation floor. Closes the serialization open-question (0008 / spec §4); four sub-items stay open (deterministic-CBOR profile; angle value type; one-vs-two channels; device COSE verification ↔ org/0006 §7 root-of-trust). Engine stays Rust (0010).
- **ADR-0012** — *Decision:* validation-driven F1 refinements from first contact with the Wolfram oracle: (1) the netz/shkia **event** uses the **50′ (0.8333°) sun-center** standard depression (refraction + solar-radius bundled, matching Wolfram `ReferenceAltitude`) via a `reference_altitude` knob — refines 0006 (Bennett stays the general/selectable refraction model); (2) solar position computed at **TT = UT + ΔT** (≈+69 s for 2026; time-varying ΔT model = TODO) — notes 0003 that low-precision NOAA vs a ±1-min bar is marginal at high latitude near the solstice; (3) elevation = the **core-domain/0004 DEM/terrain-profile path** (Copernicus **GLO-30** reconfirmed), NOT a core point-dip — the F1 core computes the sea-level astronomical horizon. *Rationale:* match the neutral oracle's published convention; knobs not branches; elevation correctness belongs to the terrain path, not a one-point dip fit. *Consequence:* low-elevation netz/shkia validate to ±1 min; elevated-horizon (Jerusalem 754 m) DEFERRED to 0004; high-lat-solstice depression precision flagged OPEN (0003). **⚠ SUPERSEDED by core-domain/0013:** the "50′-flat / defer-elevation" optics and the "precision-series" framing were based on a misread of the apparent −1.144° and were corrected by direct oracle measurement.
- **ADR-0013** — *Decision (oracle-measured; supersedes 0012's optics):* netz/shkia = apparent upper-limb at the dipped horizon (`apparent = geometric + Bennett`; target = −(semidiameter + dip)); elevation via a **`HorizonMode` knob** {Mishor | Visible (in-core dip) | TerrainProfile (0004)}; depression shitot = **geometric** sun-center (refraction-off, classical/halachic), fixture depression rows regenerated geometric; refraction-application a selectable knob; **solar series unchanged** (RA/Dec validated accurate to 0.006° vs Wolfram); type-safe Rust refactor (GeometricAltitude vs ApparentAltitude newtypes, enum refraction/horizon knobs, geometry/optics/events/params layering) + begin the 0009 parameter-vector types. *Rationale:* match the neutral oracle where physical (horizon), use the classical geometric definition where the oracle is non-physical (below-horizon refraction is a Wolfram artifact), expose every convention as an auditable parameter (halachic determinism; core resolves none), and make the apparent/geometric bug class unrepresentable. *Consequence:* F1 expected green ±1 min after implementation; no_std + 0010 exact determinism preserved.
- **ADR-0014** — *Decision:* F3 (Hebrew calendar) = **Dershowitz–Reingold fixed arithmetic** (exact integer — RD pivot, Metonic leap, molad-elapsed-days with the four dechiyot, year-types, month lengths incl. Adar I=m12/Adar II=m13, Gregorian↔Hebrew); festivals/fasts/RC as structural tokens; **`AdarAnniversaryRule` knob** (Adar II default / Adar I / Both) for common-Adar yahrzeit in a leap year; oracle = **Wolfram + Hebcal** cross-check (build/test only). *Rationale:* the post-Hillel-II calendar is fixed arithmetic — exact and authoritative, no precision/tolerance question; determinism is structural (no floats → /0010 FP gate N/A); dual oracle guards dechiyot/Adar edges; conventions are knobs (core resolves none). *Consequence:* 23/23 calendar vectors green (triple agreement on Adar I/II); core-first scope (defer parsha+yom-tov-sheni, Omer, learning cycles, tekufa/tal-u-matar, precise molad-moment→Phase 2 KL); F3↔F1 day-roll coupling = Phase 3.
- **ADR-0019** — *Decision (layering ruling, no engine code):* the multi-axis halachic **sunrise/zmanim spectrum is SECOND-ORDER** — parameters/presets over first-order outputs (ADR-0002) in the cloud config/template layer (ADR-0009 §b), on the management side of the org/0004 seam — **not** in the deterministic core, which stays convention-free *mechanism*. *Proof:* `tests/second_order.rs` reaches named positions over the **unchanged** engine — Sh"A HaRav's "all at Jerusalem 800 m" = passing `elev=800`; ChaiTables' community-earliest = `min` over reads; mishor/visible = a `horizon_mode` pick. The **only** genuine first-order gaps (deferred) are three convention-free mechanisms — `solar.limb_reference` (the beginning-vs-end-of-ascent ±semidiameter, Ish Matzliach/Yalkut Yosef vs majority), the `fixed-minute-offset` read, the `zman_definitions` decode (Phase 4b). Validation = a luach panel (mishor↔MyZmanim/OU, visible↔ChaiTables, Sephardi↔royzmanim, Chabad↔KosherJava, Jerusalem↔Tukachinsky); raw astronomy stays Wolfram-only. *Consequence:* **no knobs added to the core**; presets/area-policy/posek→vector mapping/provisioning are management-layer work. Corrects a draft that wrongly proposed pushing them into the engine.
- **ADR-0020** — *Decision (mechanism-completeness dual of 0019):* close the **three** first-order gaps 0019 §4 flagged, and only those, so the first-order read vocabulary is **total**. (1) **`LimbReference {Upper,Center,Lower}`** on `Optics` — the netz-definition (beginning-vs-end-of-ascent) dispute as a `±semidiameter` shift on the horizon target `−(limb_sign·sd + dip)`, threaded through `optics.rs`/`events.rs` incl. the terrain path; **default `Upper` is byte-identical to pre-0020**. (2) **`ReadSpec::FixedMinuteOffset{base,offset_min,seasonal}`** — the missing fifth read-spec variant (MA-72/R"T-72; `None`=fixed clock minutes, `Some((s,e))`=zmaniyos minutes over that span, reusing the `Proportional` machinery). (3) **`wire::decode_read_spec`** + integer-keyed `ReadSpecWire`/`BoundWire` (CDDL `docs/spec/read-spec.cddl`) — the engine decodes **one** read; the `zman_definitions` **catalog stays second-order** (the deliberate, narrower-than-0018-flag cut). `solar_position_reference` stays the one **baked** axis (`Unimplemented`, not faked). *Rationale:* with these, every named posek position composes from primitives — no future shita forces a core branch (0002/0008), while the core grows no policy. *Consequence:* golden 66/66 + regression **byte-unchanged (no re-bless)**, wire 5→7, read_vocab +4, **FP-determinism 536→659** native==wasm; clippy/fmt/`no_std` wasm green. *Open:* device COSE verification (↔ org/0006 §7); catalog/`obligation_sense`/rounding stay management-side; Israel DTM (0004) unaffected.
- **ADR-0018** — *Decision:* build the **reader side** of the 0011 serialization seam and **activate the TerrainProfile moat**. Reader = **`minicbor`** (no_std, **no-alloc** structured decode; links in the freestanding wasm build). Profile = **CDE / RFC 8949 §4.2.1 core-deterministic, NOT dCBOR**; numerics = **fixed-point integers** (milliarcminutes / microdegrees / mm → float-free wire) — *closing ADR-0011 sub-items 1 & 2*. `src/wire.rs` decodes a **`ParameterVector`** (retiring `Optics::default()`-only) and a borrowed zero-copy **`HorizonProfile`** (packed `i32`-LE angle byte string + `check_binding` for the 0004/0006 invariant), all failures a typed `DecodeError` (no panic). **TerrainProfile activated** via `geometry::solar_azimuth_deg` + `events::terrain_horizon_crossing` (azimuth-dependent target `angle − semidiameter`, own scan/bisect with the /0017 geometric-slope gate; scalar path byte-unchanged). Two CDDL contracts in `docs/spec/`. **COSE verification deferred** (sub-item 4 ↔ org/0006 §7 root-of-trust). *Findings:* terrain-skyline **sign** (bug caught by the differential — mountains *delay* sunrise); refraction-model + horizon_mode + fixed-behaviour **spec↔engine gaps** (flagged `Unimplemented`, not faked). *Consequence:* wire 5/5, terrain differential green, decoder fuzz (100k, no panic), regression +4 terrain rows, **FP-determinism 456→536** native==wasm; F1 66 / F2 11 / F3 38 / couplings / properties / cross-engine unchanged. *Open:* COSE/root-of-trust, channels, the spec gaps, `fixed-minute-offset`/zman-catalog (Phase 4b), Israel DTM (unaffected).
- **ADR-0017** — *Decision:* layer a robustness suite over the complete engine — **property/invariant** (round-trips, synodic/tekufa spacing, zman ordering, day-roll metamorphic), **fuzz-for-panic** (~125k adversarial calls on the FFI + readers; `no_std` panic = device hang), **determinism breadth** (native==wasm exact grid 20→**456**), **golden regression snapshot** (116 rows, `BLESS_SNAPSHOT` regen), **cross-engine differential vs KosherJava** (independent NOAA engine; sea-level sunrise/sunset as committed fixture; LGPL jar never vendored), **offline-autonomy** (10-day chag+Shabbat span, no network), and a **provisional** CI workflow (does not ratify Actions — #9 open). *Rationale:* oracle vectors prove correctness at points; properties/fuzz/regression/cross-engine prove it across the domain, guard the never-silently-wrong + never-panic invariants, and lock against drift. *Bugs found+fixed:* (1) **high-elevation horizon-crossing misfire** on the visible-sunrise/moat path — `find_crossing` judged rising/setting from the *effective* (refracted) slope, which Bennett-below-horizon flips at large dips; now judged from the **geometric** slope (regression-safe; golden 66/66 + FP unchanged); (2) **far-future overflow panic** in `shift_days` (beyond the ~2262 CE i64-nanos limit) → **saturating** add. *Consequence:* properties 10/10, fuzz 3/3, regression 116/116, offline 1/1, cross-engine **48/48 (≈2.1 s max residual)**, FP **456/456**; F1 66 / F2 11 / F3 38 / knobs 3 unchanged. *Open:* input-domain guarding, cross-engine elevation/depression scope, offline `N`, CI provider, the 2262 CE horizon.
- **ADR-0016** — *Decision:* implement the four ADR-0001 couplings (Phase 3, scope A). **(1) Day-roll** `hebrew_date_at_instant` rolls the Hebrew date at a caller-supplied F1 boundary `ReadSpec` (shkia default; bein-hashmashot safek unresolved), robust to longitude. **(2) Day-type** `classify_day(date, realm) -> DayClass` (co-holding flags), pure F3, `Realm` gating diaspora Yom Tov Sheni — the only realm branch (/0002). **(3) Full Kiddush Levana** `kiddush_levana_sayable_at` = molad window ∩ F2 moon-up ∩ F1 night (tzeit a caller knob); day-type exclusions kept at the **edge** (minhag-contested). **(4) tal-u-matar** = realm-selected **F3-class arithmetic, both branches** — EY 7 Cheshvan, diaspora 60th day after **Shmuel's** Tekufat Tishrei (new `tekufa` module, exact in regaim, 18:00 counting-roll = the Dec-5/Dec-6 mechanism). `molad_instant` factored through a shared `chalakim_to_instant`. New knobs `Realm`/`TalUmatarBasis`/`TekufaMethod`. *Rationale:* one module owns the only F1↔F3 dependency (acyclic DAG); the tekufa anchor is **calibrated, not fitted** (Julian-locked length exact, only the integer phase pinned to the published Oct-7 / Dec-4-5-6 dates + the Birkat-Hachama-2009 cross-check); contested conventions stay knobs/edge; does-not-occur stays typed, never guessed. *Finding:* tal-u-matar is **not F1 astronomy** — spec §3's "F1class.tekufa" label corrected. *Consequence:* couplings 3/3 + tekufa 10/10 (tal-u-matar Dec 5/5/6 exact) + F3 **38/38** (incl. the MyZmanim-confirmed Yom-Tov-Sheni divergence); FP-determinism **20/20** native==wasm (+kinds 11–14); F1/F2/F3 regressions byte-unchanged. **Open gates (not decided):** molad meridian; bein-hashmashot sanctity default; realm geography (a provisioned input); high-latitude fallback; /0003 tolerance; non-circularity line; Rav-Ada anchor unvalidated.
- **ADR-0015** — *Decision:* F2 (lunar) = **Meeus abridged ELP-2000/82** (ch. 47, validated exactly vs Meeus example 47.a) with **mandatory topocentric parallax** (ch. 40); moonrise/moonset **reuse the F1 crossing engine** via a `Body {Sun,Moon}` parameter (Sun path byte-unchanged) against the shared `−(semidiameter + dip)` target with a **distance-dependent moon semidiameter**; **molad-moment = exact-integer chalakim** (D–R; 765433/synodic month) pinned by the canonical **BaHaRaD** anchor, projected to UT via a **flagged meridian constant** (the one float step); **Kiddush Levana** window = molad + `KiddushLevanaStart` (ThreeDays default / SevenDays / Molad) + `KiddushLevanaEnd` (HalfMonth default / FifteenDays) knobs, with the **night ∩ moon-up ∩ window intersection deferred to Phase 3**; oracle = **Wolfram `MoonPosition`** (+ BaHaRaD/Hebcal for molad). *Rationale:* validate the new physics (ELP series + parallax) to arc-seconds against a neutral oracle, then reuse the proven crossing path (one tested code path); the molad is exact fixed arithmetic anchored on the universally-agreed BaHaRaD, with the single UT-projection float isolated and flagged; conventions are knobs (core resolves none); the visibility intersection is a coupling, kept for the coupling phase. *Consequence:* F2 11/11 (altitude ≤19″, rise/set ±18 s vs ±60 s bar), F3 27/27 (+molad), F1 66/66 unaffected; FP-determinism 15/15 native==wasm (now incl. F2/molad probes). **Open/flagged:** molad meridian convention (UT projection only); KL 3-vs-7-days genuinely contested (knob).
